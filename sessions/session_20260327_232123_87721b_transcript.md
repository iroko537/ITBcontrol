can you spawn haiku subagents in hermes?

## ASSISTANT []
Not directly via delegate_task — that tool doesn't have a model parameter, it inherits whatever model the session is using.

Two workarounds:

1. Terminal spawn — run hermes CLI as a subprocess with a specific model:
   hermes chat -Q --provider anthropic -m claude-haiku-4-5 --yolo -q "your task"

2. Cronjob — the cron tool does have a model parameter, so you can schedule one-shot jobs with haiku:
   mcp_cronjob(action='create', model='claude-haiku-4-5', provider='anthropic', ...)

Want me to try either approach for something specific?

## USER []
how the cronjob has a model and the delegate not?

## ASSISTANT []
Good question — it's a design inconsistency in Hermes, not something I can fix.

The reasoning seems to be:

- Cron jobs run in completely fresh, standalone sessions with no parent context, so they need full configuration (model, provider, base_url) to know what to use.

- delegate_task subagents are meant to be lightweight workers that spin up within the same session infrastructure and just inherit the parent's model/provider. The model param was never added to it.

Whether that's intentional or an oversight is unclear — it's a valid gap. You could file it as a feature request if you want haiku subagents for cost-efficient parallel work. The terminal workaround (spawning hermes CLI as a subprocess) is the only real escape hatch right now.
